# Road Config Tool

Language: English | [中文](README.zh-CN.md)

A zero-dependency, purely frontend **road config visualization tool** for viewing and checking
road config annotation files used by [OpenVTER](https://github.com/XinkaiJi/OpenVTER).

After loading a road config (`.json`) file, the tool renders lane numbers, lanelines, driving-line mileage,
workzones, coordinate axes, and related regions on top of the base image. The rendered result can also be
exported as a PNG image.

This repository contains two pages:

- **Viewer** (`index.html`) - Load and visualize an existing road config.
- **Editor** (`editor.html`) - Draw or edit annotations on a base image and export a compatible road config JSON.

## Online Access

- Viewer: https://xinkaiji.github.io/road_config_website/
- Editor: https://xinkaiji.github.io/road_config_website/editor.html

No installation or backend service is required. All parsing and rendering happen locally in your browser, and
annotation files are never uploaded to a server.

## What This Is

[OpenVTER](https://github.com/XinkaiJi/OpenVTER) uses a road config JSON file to describe road geometry and
semantic information, including lanes, lanelines, driving mileage lines, coordinate axes, and workzones. This
JSON is usually generated by an annotation tool with a labelme-style `shapes` structure.

This tool mirrors the drawing logic of OpenVTER's `visualize_config`, so you can quickly preview annotation
results without running the Python script.

## Viewer Features

- **Loading**
  - Click "Open road config" and choose a `.json` file.
  - Drag a `.json` file directly onto the canvas.
  - If the JSON contains embedded `imageData` as base64, the Viewer uses it automatically. Otherwise, you can
    choose a separate base image.
- **Visualization layers** with independent toggles and count indicators:
  - Base image with adjustable brightness.
  - Lanes built from adjacent `laneline` pairs plus closed `lane_N` regions, with lane numbers.
  - Lanelines (`laneline`).
  - Driving lines and mileage labels, sampled at the configured interval.
  - Coordinate axes (`x` / `y`) with ticks and meter labels.
  - Workzones (`workzone`) highlighted with orange dashed outlines.
  - Detection, stable, and intersection regions (`fp` / `road` / `int`).
  - Text label visibility.
- **Interaction**
  - Mouse wheel zooms around the cursor.
  - Right-drag or middle-drag pans the canvas.
  - One-click "Fit" and "Reset view".
- **Parameters**: base image brightness, lane fill opacity, font size, line width, and mileage interval.
- **Export**: save the current rendering as PNG.

## Editor Features

`editor.html` creates or edits road configs from scratch. Exported JSON follows the labelme-style structure
with `shape_type`, `group_id`, and embedded base image data, and can be opened directly in the Viewer or used
by OpenVTER.

- **Loading**: open a base image, or load an existing `.json` to continue editing. Both images and JSON files
  support drag-and-drop.
- **Rule-based annotation**: choose an annotation type from the dropdown, and the editor generates the correct
  `label` and geometry type automatically:
  `road` / `fp` (polygon or rectangle) / `length_X` / `x` / `y` / `lane_N` / `laneline_N` /
  `drivingline_{name}_{pos}_region|line|point_{base}|connect_{id1}_{id2}` / `workzone`.
  `lane_N` and `laneline_N` numbers auto-increment after each completed annotation.
- **Custom annotations**: choose "Custom", select a shape type (point, line, rectangle, polygon, or polyline),
  then enter any `label` after drawing, similar to labelme.
- **Draw mode** (`B`): left-click to add points, double-click or press Enter to finish polygons/polylines,
  Backspace removes the last point, and Esc cancels. Lines finish after two points; rectangles use click-drag;
  points use a single click.
- **Select / edit mode** (`V`): click to select, drag vertices to adjust, drag the whole shape to move,
  press Delete to remove, and double-click a list item to rename its label.
- **View controls**: mouse wheel zoom, right-drag or middle-drag pan, and base image brightness adjustment.
- **Export**: road config `.json` or annotated PNG.

## Road Config Format

The JSON top level contains `imageWidth`, `imageHeight`, optional `imageData` (base64 base image), and a
`shapes` array. Each shape contains `label` and `points`, and is interpreted by label prefix:

| Label prefix       | Meaning                                           |
| ------------------ | ------------------------------------------------- |
| `lane_N`           | Closed region for lane N                          |
| `laneline_N`       | Lane line; adjacent numbers are paired as lanes   |
| `drivingline*`     | Driving line used for mileage calculation         |
| `length_<meters>`  | Scale annotation: segment real-world length       |
| `x` / `y`          | Coordinate axis direction                         |
| `workzone*`        | Workzone                                          |
| `fp`               | Stable / detection region                         |
| `road`             | Road surface region                               |
| `int*`             | Intersection region                               |

Scale in meters per pixel is inferred from `length_*` annotations and used for mileage and coordinate-axis
unit conversion.

## Local Use

This is a static site with no build step. Open `index.html` directly in your browser:

```bash
git clone https://github.com/XinkaiJi/road_config_website.git
cd road_config_website
open index.html        # macOS; or open this file directly in your browser
```

## Deployment

The site is hosted with GitHub Pages. In repository **Settings -> Pages**, set the source to the `main` branch
root directory.
